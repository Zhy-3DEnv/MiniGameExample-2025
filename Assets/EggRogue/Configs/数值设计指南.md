## 数值设计核心原则

### 1. **玩家体验优先**
- **目标**：让玩家感受到成长、挑战、成就感
- **避免**：数值爆炸、完全碾压、完全打不过

### 2. **渐进式难度**
- **前期**：简单，让玩家熟悉机制
- **中期**：逐步增加挑战
- **后期**：高难度但仍有希望

### 3. **可感知的成长**
- 每次升级/选卡/购买都能明显感受到变强
- 数值变化要足够大，让玩家能感受到（建议至少 10-20% 的提升）

### 4. **风险与收益平衡**
- 高风险高收益（例如：高难度关卡给更多金币）
- 低风险低收益（例如：简单关卡给基础奖励）

## 肉鸽游戏数值设计思路

### 核心循环
```
玩家开始 → 战斗 → 获得资源 → 成长（选卡/商店）→ 更强 → 挑战更高难度 → 循环
```

### 关键数值关系

#### **DPS（每秒伤害）计算**
```
玩家DPS = 基础伤害 × 攻击速度 × 伤害倍率（卡片/道具）
怪物DPS = 怪物伤害 × 攻击频率（通常为接触伤害）
```

#### **生存时间估算**
```
击杀一个怪物需要时间 = 怪物血量 / 玩家DPS
玩家能承受的怪物数量 = 玩家血量 / (怪物DPS × 怪物数量)
```

#### **难度评估**
```
关卡难度 = (怪物数量 × 怪物血量 × 怪物移速) / (玩家DPS × 玩家移速 × 玩家血量)
```

## 关卡难度曲线设计

### 难度增长公式（推荐）

#### **方案A：线性增长（适合新手）**
```
第N关难度 = 基础难度 × (1 + (N-1) × 0.1)
例如：第1关=1.0，第2关=1.1，第3关=1.2，第10关=1.9
```

#### **方案B：指数增长（适合硬核）**
```
第N关难度 = 基础难度 × (1.15 ^ (N-1))
例如：第1关=1.0，第2关=1.15，第3关=1.32，第10关=3.05
```

#### **方案C：分段增长（推荐，平衡）**
```
前5关：线性增长（让玩家适应）
第6-15关：指数增长（挑战增加）
第16关+：缓慢增长（避免数值爆炸）
```

### 你的关卡配置建议

#### **关卡时长设计**
- **第1关**：20秒（教学关，让玩家熟悉）
- **第2-5关**：25-30秒（逐步增加）
- **第6-10关**：30-40秒（挑战增加）
- **第11关+**：35-45秒（高难度）

#### **刷怪频率设计**
```
第1关：5个/秒（快速上手）
第2-3关：3-4个/秒（稳定节奏）
第4-6关：2-3个/秒（增加难度）
第7关+：1.5-2个/秒（高难度，但给玩家反应时间）
```

#### **难度倍率设计（参考你现有的）**
```
第1关：血量×1.0，移速×1.0，金币×1.0
第2关：血量×1.05，移速×1.02，金币×1.03
第3关：血量×1.1，移速×1.04，金币×1.06
第4关：血量×1.15，移速×1.06，金币×1.09
第5关：血量×1.2，移速×1.08，金币×1.12
...
第10关：血量×1.5，移速×1.2，金币×1.3
```

**建议公式**：
- **血量倍率**：`1 + (关卡数-1) × 0.05`（每关+5%）
- **移速倍率**：`1 + (关卡数-1) × 0.02`（每关+2%）
- **金币倍率**：`1 + (关卡数-1) × 0.03`（每关+3%，让玩家有成长感）

## 角色平衡设计

### 角色定位分类

#### **1. 均衡型（推荐作为默认角色）**
```
血量：100
伤害：10
攻速：2.0/秒
移速：5
攻击范围：10
```
**特点**：各项属性平衡，适合新手

#### **2. 高伤害型（玻璃大炮）**
```
血量：70（-30%）
伤害：15（+50%）
攻速：2.5/秒（+25%）
移速：6（+20%）
攻击范围：12（+20%）
```
**特点**：高输出，但容错率低

#### **3. 坦克型（高生存）**
```
血量：150（+50%）
伤害：8（-20%）
攻速：1.5/秒（-25%）
移速：4（-20%）
攻击范围：8（-20%）
```
**特点**：高血量，但输出较低

#### **4. 速度型（高机动）**
```
血量：80（-20%）
伤害：9（-10%）
攻速：2.0/秒
移速：7（+40%）
攻击范围：10
```
**特点**：高移速，适合走位

### 角色平衡检查清单

✅ **每个角色都应该有：**
- 独特的玩法体验
- 明确的优势（至少1-2项属性突出）
- 明确的劣势（至少1-2项属性较弱）
- 在特定关卡/情况下有优势

❌ **避免：**
- 某个角色完全碾压其他角色
- 某个角色完全没用
- 所有角色数值过于接近（没有差异化）

## 怪物平衡设计

### 怪物类型分类

#### **1. 基础怪（最常见）**
```
血量：20
移速：2-3
伤害：1-2
掉落：1-2金币
```
**作用**：提供基础威胁和资源

#### **2. 快速怪（高威胁）**
```
血量：15（-25%）
移速：4-5（+67%）
伤害：1-2
掉落：1-2金币
```
**作用**：增加压力，考验玩家反应

#### **3. 坦克怪（高血量）**
```
血量：50（+150%）
移速：1.5（-25%）
伤害：2-3（+50%）
掉落：2-3金币
```
**作用**：需要集中火力，考验DPS

#### **4. 精英怪（高价值）**
```
血量：40（+100%）
移速：3
伤害：3（+50%）
掉落：5-8金币
```
**作用**：高风险高收益，提供挑战

### 怪物平衡检查

✅ **每个怪物类型应该：**
- 有明确的威胁点（高血量/高移速/高伤害）
- 有明确的弱点（低移速/低血量/低伤害）
- 掉落与难度成正比

❌ **避免：**
- 怪物移速过快导致玩家无法躲避
- 怪物血量过高导致战斗拖沓
- 怪物伤害过高导致玩家瞬间死亡

## 经济系统设计

### 金币获取来源

#### **1. 击杀怪物掉落**
```
基础怪：1-2金币
快速怪：1-2金币
坦克怪：2-3金币
精英怪：5-8金币
```

#### **2. 关卡胜利奖励**
```
第1关：10金币（基础）
第2关：12金币（+20%）
第3关：15金币（+25%）
第4关：18金币（+20%）
...
第10关：30金币
```

**建议公式**：`10 + (关卡数-1) × 2` 或 `10 × (1.15 ^ (关卡数-1))`

### 金币消耗设计

#### **商店价格建议**
```
武器：50-200金币（根据等级）
道具：30-100金币（根据效果）
刷新：10-20金币（递增）
```

### 经济平衡检查

✅ **玩家应该能够：**
- 每关至少购买1-2件商品
- 通过多关积累购买高级商品
- 感受到金币的价值（不能太容易获得）

❌ **避免：**
- 金币过多导致商店失去意义
- 金币过少导致玩家无法成长
- 价格过高导致玩家无法购买

## 测试与迭代方法

### 1. **建立测试基准**

#### **目标体验**
- 第1关：玩家应该轻松通过（90%+通过率）
- 第5关：玩家应该感到有挑战但仍能通过（60-70%通过率）
- 第10关：玩家应该感到困难但仍有希望（30-40%通过率）

#### **关键指标**
- **TTK（Time To Kill）**：击杀一个怪物需要的时间
  - 基础怪：1-2秒
  - 坦克怪：3-5秒
  - 精英怪：2-4秒

- **生存时间**：玩家在怪物包围下能坚持多久
  - 第1关：玩家应该能轻松生存
  - 第5关：玩家需要走位和策略
  - 第10关：玩家需要完美操作

### 2. **数值测试流程**

#### **步骤1：理论计算**
```
计算玩家DPS = 基础伤害 × 攻速
计算怪物血量 = 基础血量 × 关卡倍率
计算TTK = 怪物血量 / 玩家DPS
```

#### **步骤2：实际测试**
- 用默认角色测试每关
- 记录：通关时间、死亡次数、剩余血量、获得金币
- 调整数值直到体验符合预期

#### **步骤3：不同角色测试**
- 用每个角色测试相同关卡
- 确保没有角色过于强势或弱势

### 3. **迭代调整原则**

#### **如果玩家觉得太简单**
- 增加怪物血量（+10-20%）
- 增加怪物移速（+5-10%）
- 增加刷怪频率（+10-20%）

#### **如果玩家觉得太难**
- 减少怪物血量（-10-20%）
- 减少怪物移速（-5-10%）
- 减少刷怪频率（-10-20%）
- 增加玩家基础属性（+5-10%）

#### **如果玩家觉得无聊**
- 增加怪物类型多样性
- 增加关卡时长
- 增加特殊事件/机制

## 总结

### 核心原则
1. **渐进式难度**：每关比上一关难10-15%
2. **可感知成长**：每次成长至少提升10-20%
3. **风险收益平衡**：高难度=高奖励
4. **持续测试**：理论计算+实际测试+迭代调整

### 下一步行动
1. ✅ 调整关卡胜利奖励（第3关6→15金币）
2. ✅ 补充怪物类型（坦克怪、精英怪）
3. ✅ 测试每关的通关体验
4. ✅ 根据测试结果微调数值

### 推荐工具
- **Excel表格**：记录每关的数值配置和测试结果
- **Unity Profiler**：监控性能，确保数值不会导致卡顿
- **玩家测试**：找朋友测试，收集反馈

---

**记住**：数值设计是一个迭代过程，没有完美的数值，只有不断优化的数值。多测试、多调整，直到玩家体验达到最佳！
